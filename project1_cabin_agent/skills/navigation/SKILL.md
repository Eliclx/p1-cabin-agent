# Navigation Domain Knowledge

> 读者：云端 LLM（端侧无法处理时的兜底层）
> 定位：域知识补充，不替代 schema.py（schema 是唯一真相源）
> 时机：端侧降级 / API 失败 / 复杂多轮对话时，云端读取此文件

---

## 域边界

你负责处理以下用户意图，其他域不要接：

| 用户输入示例 | 归你 | 原因 |
|---|---|---|
| "导航到春熙路" | ✓ | 明确导航 |
| "附近有没有加油站" | ✓ | 周边搜索属于导航域 |
| "加油站还有多远" | ✓ | 需要距离计算 |
| "还有多久到" | ✓ | 查询当前导航 ETA |
| "换条路走" | ✓ | 重新规划，route_type=shortest |
| "取消导航" / "不去了" | ✓ | 取消当前导航 |
| "现在外面几度" | ✗ chitchat | 天气 |
| "油量还有多少" | ✗ vehicle | 车况 |
| "播放导航语音" | ✗ media | 媒体 |

歧义 case：
- "附近有好吃的" → search_nearby, keyword=餐厅（不是 chitchat）
- "这里停车方便吗" → search_nearby, keyword=停车场（不是闲聊）
- "找最近的加油站过去" → 两个任务：search_nearby → navigate_to（需要依赖链）

## 坐标格式

所有坐标严格使用高德坐标系（GCJ-02），格式 `经度,纬度`，例如 `104.07,30.65`。
不要使用反向顺序或度分秒格式。

## 云端兜底指令

### API 不可用时的回复策略

当导航工具调用失败时：
- 若有 L2 行程记忆中的 last_destination → "导航服务暂时不可用，为您继续上次的目的地{last_destination}"
- 无缓存 → "导航服务暂时不可用，请稍后重试"
- 不要向用户暴露"高德 API""高德地图"等技术术语，统一说"导航服务"

### L3 未配置时的回复策略

用户说"回家"/"去公司"但用户偏好中没有配置对应地址时：
- "您还没有设置{家/公司}的地址，请先在设置中添加"
- 不要自行猜测或编造地址

### Harness 需要确认时

当系统返回 need_confirm 信号时（如高速行驶中改目的地、超长距离确认）：
- 直接向用户复述确认信息，不要修改措辞
- 用户确认 → 执行；用户拒绝 → 取消，回复"好的，已取消"

### 搜索无结果时

周边搜索返回空结果：
- "附近没有找到{keyword}，您可以试试扩大搜索范围或换个关键词"
- 不要编造不存在的地点

## 别名解析规则

以下语义别名由 Harness 层自动处理，你不需要查询地址：

| 用户说 | Harness 行为 | 如果解析失败 |
|---|---|---|
| "家" / "回家" | 从 L3 用户偏好取 home_address | 追问用户设置 |
| "公司" / "单位" | 从 L3 用户偏好取 company_address | 追问用户设置 |
| "上次去的" / "刚才那里" | 从 L2 行程记忆取 last_destination | 告知无记录 |

你只需要在输出中保留原始别名（如 destination="家"），底层会自动替换。

## 未定义 Intent

以下用户意图暂未在 schema 中定义，按 direct_answer 处理：

- "取消导航" / "不去了" → 取消当前导航，回复"好的，已取消导航"
- "还有多久到" → 如果有活跃导航，从 L2 取 ETA 信息回复；如果没有，回复"当前没有进行中的导航"
